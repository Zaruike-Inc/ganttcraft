# Security Policy

## Reporting Security Issues

If you discover a security issue in this project, please inform us immediately. We take all potential vulnerabilities seriously and will work with you to quickly and effectively resolve the problem.

To report a security issue, please send an email to [our security team](mailto:security@zaruike.dev). Do not create a public issue for security problems, as this could potentially expose vulnerabilities to others.

## Reporting Process

1. Send an email to our security team at [security@zaruike.dev](mailto:security@zaruike.dev). Include as many details as possible about the vulnerability.

2. Our security team will review your report and respond to you promptly to discuss additional details and next steps.

3. Once the vulnerability is confirmed, we will work to resolve it promptly. If necessary, we may ask you not to publicly disclose the vulnerability until it is fixed.

4. Once the vulnerability is fixed, we will release an updated version of the software and credit you for the discovery if you wish.

## Scope

Please note that this security policy applies to vulnerabilities specific to the software of this repository. For other issues or questions, please contact us through the appropriate channels indicated in the README file.

We greatly appreciate your assistance in maintaining the security of this project. Thank you for your collaboration.
